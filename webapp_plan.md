# 생기부 분석 웹 애플리케이션 기획서

## 1. 프로젝트 개요

**목적**: 담임교사가 학생 28명의 학교생활기록부(생기부) 데이터를 한눈에 파악하고,
AI를 활용해 활동 요약·진로 조언·면접 자료를 자동 생성하는 웹 도구.

**데이터 소스** (7개 파싱 완료 JSON)

| # | 파일 | 핵심 데이터 | 주요 필드 |
|---|------|------------|----------|
| 01 | 학년이력 | 학년·반·번호·담임 | `entries[].school_grade, class_no, student_no, homeroom_teacher` |
| 02 | 출결상황 | 학년별 출석·결석·지각 | `records[].school_days, absence, late, early_leave, note` |
| 03 | 창의적체험활동 | 자율/동아리/진로활동 | `activities[].domain, hours, details` |
| 04 | 봉사활동 | 봉사기관·시간·누적 | `entries[].organization, activity, hours, cumulative_total` |
| 05 | 교과학습발달 | 과목별 원점수/평균/등급/석차 | `grades[].subject_name, score{raw,avg,stddev}, rank_grade, achievement` |
| 06 | 세부능력및특기사항 | 과목별 세특 텍스트 | `notes[].by_subject[].subject, text` |
| 07 | 행동특성및종합의견 | 학년별 담임 종합 의견 | `comments[].comment` |

## 2. 페이지 구조

### 2-1. 학생 목록 (메인 대시보드)

| 요소 | 설명 |
|------|------|
| 학생 카드/테이블 | 28명 전체 리스트. 이름·번호·현재 학년 표시 |
| 퀵 요약 배지 | 출결(개근 여부), 최근 등급 평균, 봉사시간 합계 등 한줄 뱃지 |
| 검색/필터 | 이름 검색, 학년 필터 |
| 클릭 → 학생 상세 페이지 | |

### 2-2. 학생 상세 페이지 (탭 구조)

```
[기본정보] [성적] [출결] [창체] [봉사] [세특] [행특] [AI 분석] [면접준비]
```

#### 탭 ① 기본정보
- 01_학년이력: 학년별 반·번호·담임 테이블
- 현재 학년/반/번호 강조 표시

#### 탭 ② 성적 (시각화 핵심)
- **데이터**: 05_교과학습발달상황
- **차트 1 — 학년별 등급 추이** (Line Chart)
  - X축: 학기(1-1, 1-2, 2-1, 2-2, 3-1)
  - Y축: 석차등급(1~9, 역순)
  - 계열: 과목군별 또는 주요 과목별 선택 가능
- **차트 2 — 과목별 원점수 vs 평균 비교** (Grouped Bar)
  - 학기 선택 → 해당 학기 과목들의 (본인 원점수 / 과목평균) 비교
- **차트 3 — 과목군 레이더** (Radar Chart)
  - 국어/수학/영어/사회/과학/예체능 등 과목군별 등급을 방사형으로
- **성적 요약 테이블**
  - 전 학기 과목별 등급/원점수/평균/표준편차/성취도 일람표

#### 탭 ③ 출결
- **데이터**: 02_출결상황
- 학년별 출석일수, 결석/지각/조퇴 테이블
- 결석 특이사항(note) 강조 표시
- 개근 학년 뱃지

#### 탭 ④ 창의적체험활동
- **데이터**: 03_창의적체험활동상황
- 영역별(자율·동아리·진로) 카드 뷰
- 학년별 시간 합산 표시
- 활동 상세 텍스트 펼치기/접기(Accordion)

#### 탭 ⑤ 봉사활동
- **데이터**: 04_봉사활동상황
- 타임라인 뷰: 날짜순 봉사 이력
- 기관별 그룹핑 가능
- 누적 시간 차트(학년별 Bar)

#### 탭 ⑥ 세부능력및특기사항
- **데이터**: 06_세부능력및특기사항
- 학년 선택 → 과목별 세특 텍스트 카드
- 과목 태그 필터
- 키워드 하이라이트 검색

#### 탭 ⑦ 행동특성및종합의견
- **데이터**: 07_행동특성및종합의견
- 학년별 의견 전문 표시
- 학년 비교 보기(좌우 또는 타임라인)

## 3. AI 기능 설계

> **공통**: Google Gemini API (gemini-2.5-flash 등) 사용.
> 프론트에서 API Key를 입력받아 브라우저에서 직접 호출하거나,
> 간단한 백엔드 프록시를 거치는 방식 중 선택.

### 3-1. AI 활동 요약 (탭 ⑧ "AI 분석")

**목적**: 학생의 전 영역 데이터를 종합하여 한눈에 파악 가능한 요약문 생성

**입력 (프롬프트에 포함할 데이터)**:
- 05: 주요 과목 등급 추이 (요약 테이블)
- 06: 과목별 세특 핵심 문장
- 03: 창체 활동 내역
- 04: 봉사 이력
- 07: 행특 종합의견

**출력 구조**:
```
1. 학업 역량 요약 (성적 추이 + 세특 기반)
2. 핵심 활동/경험 키워드 (최대 10개)
3. 강점 분석 (3~5개 항목)
4. 성장 스토리 (학년별 변화 서술)
5. 보완 필요 영역 (부드럽게 서술)
```

### 3-2. AI 진로·진학 조언 (탭 ⑧ "AI 분석" 하위)

**입력**:
- 위 요약 데이터 + 교사가 입력하는 추가 정보:
  - 학생 희망 진로/학과 (텍스트 입력)
  - 목표 대학 유형 (수시/정시, 학생부교과/종합 등)

**출력 구조**:
```
1. 진로 적합성 분석
   - 희망 학과와 생기부 데이터 간 연결점
   - 부족한 연결고리 및 보완 방안
2. 추천 학과/계열 (생기부 데이터 기반 3~5개)
3. 학생부종합전형 대비 전략
   - 자소서 소재 후보 (세특/창체에서 추출)
   - 강조할 활동 & 스토리라인
4. 남은 학기 활동 제안
```

### 3-3. AI 면접 자료 생성 (탭 ⑨ "면접 준비")

**입력**:
- 희망 학과 (교사 입력, 필수)
- 대학명 (선택)
- 면접 유형: 일반면접 / 서류기반면접 / 제시문면접
- 생기부 데이터 전체 (자동 주입)

**출력 구조**:
```
[서류 기반 질문] (5~10개)
  Q1. (세특/창체 내용에서 추출한 구체적 질문)
      → 모범 답안 (생기부 내용 기반 + 학과 연결)
      → 꼬리 질문 예시 1~2개
  Q2. ...

[학과 적합성 질문] (3~5개)
  Q1. (지원 학과의 핵심 주제 관련)
      → 모범 답안 (생기부 활동과 연결하여)
  Q2. ...

[인성/가치관 질문] (2~3개)
  Q1. (행특/봉사/창체에서 추출)
      → 모범 답안
  Q2. ...

[면접 TIP]
  - 이 학생이 특히 강조해야 할 포인트
  - 예상 약점 질문 대비 전략
```

**UX 흐름**:
1. 교사가 학과/대학/면접유형 입력
2. "면접 자료 생성" 버튼 클릭
3. 로딩 중 스트리밍으로 실시간 표시
4. 완성 후 PDF 다운로드 / 클립보드 복사

## 4. 추가 기능 아이디어

### 4-1. 학생 비교 뷰
- 2~3명 학생을 선택하여 성적/활동을 나란히 비교
- 성적 차트 오버레이

### 4-2. 학급 전체 통계
- 과목별 등급 분포 (히스토그램)
- 출결 현황 요약
- 봉사시간 분포

### 4-3. 생기부 텍스트 검색
- 전체 학생의 세특/행특/창체에서 키워드 전문 검색
- 특정 활동이나 프로그램명으로 참여 학생 빠르게 찾기

### 4-4. 내보내기
- 학생별 종합 보고서 PDF 출력
- AI 생성 자료 저장/이력 관리

## 5. 기술 스택 (제안)

| 영역 | 기술 | 이유 |
|------|------|------|
| 프레임워크 | **Next.js (React)** | SSG로 JSON 빌드타임 로드, 빠른 SPA |
| 스타일링 | **Tailwind CSS + shadcn/ui** | 모던 UI, 빠른 개발 |
| 차트 | **Recharts** 또는 **Chart.js** | React 친화, 등급/점수 시각화 |
| AI 연동 | **Google Gemini API (gemini-2.5-flash)** | 브라우저 직접 호출 or 간이 API Route |
| 아이콘 | **Lucide React** | 경량, shadcn과 호환 |
| 배포 | **Netlify** (정적) 또는 **Vercel** | JSON 내장, 서버리스 |
| PDF 생성 | **html2pdf.js** 또는 **jsPDF** | 클라이언트사이드 PDF |

**핵심 설계 원칙**:
- 7개 JSON을 빌드 시 통합하여 `students/{name}.json` 형태의 개별 파일로 변환
- 또는 하나의 `merged_data.json`으로 합쳐서 프론트에서 소비
- AI API Key는 사용자가 직접 입력 (Settings 패널), localStorage 저장
- 서버 불필요: 순수 정적 사이트 + 클라이언트사이드 AI 호출

## 7. AI 프롬프트 설계 방향

### 공통 시스템 프롬프트
```
당신은 한국 고등학교 담임교사를 보조하는 생기부 분석 전문 AI입니다.
학생의 학교생활기록부 데이터를 기반으로 객관적이고 건설적인 분석을 제공합니다.
- 항상 생기부 데이터에 근거하여 답변하세요.
- 추측이나 허위 정보를 생성하지 마세요.
- 학생의 성장 가능성을 긍정적으로 바라보되, 현실적인 조언을 제공하세요.
- 한국 대입 제도(학생부종합전형, 교과전형 등)에 대한 이해를 바탕으로 답변하세요.
```

### 토큰 최적화
- 전체 세특/행특 원문을 모두 넣으면 토큰 과다 → 각 과목별 세특은 앞 200자 요약본 사용
- 성적은 테이블 형식(CSV 스타일)으로 압축하여 전달
- 필요시 2단계 호출: ①요약 생성 → ②요약 기반 심층 분석

## 8. 데이터 보안 고려사항

- **개인정보**: 학생 실명·성적이 포함되므로 배포 시 접근 제한 필수
  - 비밀번호 잠금 (간단한 클라이언트 인증) 또는 로컬 전용
  - 외부 배포 시 `.htaccess` 또는 Netlify 비밀번호 보호
- **AI API 호출**: 학생 데이터가 Google 서버로 전송됨
  - 교사에게 이 사실을 명시적으로 고지하는 UI 삽입
  - 가능하면 요약/익명화된 데이터만 전송
